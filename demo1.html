设置 用户名 邮箱 git config --global user.name "用户名" git config --global user.email "邮箱" 克隆远程仓库的文件到本地 git clone 仓库地址 git status 查看工作区状态 git add + 文件名 将文件提交到暂存区 git add . 将全部修改过的文件存到暂存区 git commit 将暂存区的文件提交到版本库 git commit -m "注释" 将暂存区的文件提交到版本库，同时加上注释 git commit -a -m "注释" 将工作区的文件存到暂存区和版本库，并且同时加上注释 git log 查看提交记录 对比 git diff 工作区和暂存区的内容对比 git diff --cached 暂存区和版本库的内容对比 git diff master 工作区和版本库的内容对比 撤销 git reset HEAD 文件名 从暂存区撤回到工作区 git checkout -- 文件名 将工作区的内容撤回到暂存区或者版本库 git commit -m "注释" --amend 将本次提交和上一次提交到版本库的提交合并 删除 git rm --cached 文件名 在暂存区将对应的文件删掉 git rm -f 文件名 工作区和暂存区对应的文件一起删掉 git rm 文件名 在工作区将对应的文件删掉（也可直接删掉） 恢复 git checkout id 文件名 恢复某个对应的文件 git reset --hard id 恢复到对应的版本 git reset --hard HEAD^ 恢复到前一个版本 git reset --hard HEAD~3 恢复到之前指定的第3个版本 git reflog 打出所有提交过的记录，回到指定版本 git remote 查看分支名(origin) git push 分支名 master 提交代码到远程仓库 解决冲突 git fetch 发现冲突时拉取版本库中的代码（并不与我本地工作区的代码合并） git diff master 分支名/master 查看冲突内容 git merge 分支名/master 将冲突手动合并到本地工作区 git pull 发现冲突时拉取版本库中的代码（直接合并我本地的代码，省去了diff和merge） 要是我本地修改了，与版本库里的冲突了，会显示冲突内容（版本库和工作区） 没修改的话，版本库会直接覆盖我本地的内容 分支 git branch 查看分支 git branch 分支名 创建分支 git checkout 分支名 切换到对应的分支名 git checkout -b 分支名 创建一个分支，并且同时切换到这个分支 git merge 分支名 将master与这个分支合并，master指向向前移动 git branch --merged 查看合并了的分支 git branch --no-merged 查看没有合并的分支 git branch -d 分支名 删除指定的分支（合并以后的才能删，没有合并的不能删） git branch -D 分支名 强制删除没有合并的分支
